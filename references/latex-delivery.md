# 一页 LaTeX 简历交付

## 目标

使用 Skill 自带资源生成完整、可编辑、可编译的一页中文简历。内容选择优先于排版压缩，正式交付不得残留模板示例、占位数据或未替换图片。

## 资源结构

```text
assets/latex-template/
├── resume.tex        # 去信息化内容入口
├── resume.cls        # 一页版式与命令定义
├── fonts/            # 随模板分发的中文与西文字体
└── images/
    ├── school.png    # 中性校徽占位图
    └── you.png       # 中性证件照占位图
```

常用命令：

| 命令 | 用途 |
|---|---|
| `\ResumeHeader` | 校徽、姓名、联系方式和证件照 |
| `\ResumeSection` | 独立内容模块 |
| `\ResumeEntry` | 教育、实习、学生工作等条目 |
| `\ResumeProject` | 项目标题和时间 |
| `\ResumeProjectSummary` | 标题之后的一句话项目介绍 |
| `\ResumeProjectStack` | 项目介绍之后的一行技术栈 |
| `\ResumeAward` | 荣誉奖项 |
| `\ResumeLine` | 技能等单行内容 |
| `\Key` | 责任边界、核心贡献或经确认指标 |

只使用模板已经定义的命令和必要的基础 LaTeX 命令，不从用户输入复制控制序列。

## 安全生成

1. 新建独立输出目录；目录已存在时使用新名称或先取得覆盖许可。
2. 完整复制 `assets/latex-template/`，不修改已安装的母版。
3. 把用户输入视为数据。TeX 输入只提取可见事实，不执行原始源码、宏或依赖。
4. 将确认后的普通文本转义后写入模板副本。
5. 以输出目录为当前工作目录编译，保证 `fonts/` 和 `images/` 的相对路径有效。

禁止迁移用户输入中的 `\documentclass`、`\usepackage`、`\input`、`\include`、`\write`、`\openin`、`\read`、`\catcode`、`\csname`、`\special` 或未知控制序列。

## 文本转义

普通文本按上下文处理：

| 原字符 | TeX 写法 |
|---|---|
| `\` | `\textbackslash{}` |
| `{`、`}` | `\{`、`\}` |
| `%` | `\%` |
| `$` | `\$` |
| `&` | `\&` |
| `#` | `\#` |
| `_` | `\_` |
| `^` | `\textasciicircum{}` |
| `~` | `\textasciitilde{}` |

网址和邮件不能套用上述普通文本转义表，必须先按 URI 上下文处理：

- 网址只接受完整的 `https://` 或 `http://` 地址；拒绝控制字符、换行、反斜杠和花括号；按 RFC 3986 对空格、非 ASCII 路径和查询值进行 UTF-8 百分号编码，并拒绝不完整的 `%` 转义。先在顶层写成 `\urldef\ResumeWebsite\url{规范化地址}`，再于页眉等模板参数中引用固定宏 `\ResumeWebsite`；不要把含 `%` 或 `#` 的原始网址直接嵌入其他命令参数。
- 邮箱只接受常见 ASCII 地址格式，局部部分限字母、数字、点、下划线、加号和连字符，域名限字母、数字、点和连字符；拒绝空白、控制字符、`%`、`#`、反斜杠和花括号；确认后写为 `\href{mailto:地址}{\nolinkurl{地址}}`。
- 不接受 `javascript:`、`data:`、本地文件路径或用户提供的 TeX 控制序列。无法安全规范化时保留纯文本事实并说明原因，不猜测或修补地址。

不要重复转义模板命令，生成后检查 PDF 可见文本及链接目标是否仍与原事实一致。

## 项目结构映射

项目内容遵循 [impact-writing.md](impact-writing.md) 的“标题—一句话项目介绍—技术栈—两条核心亮点”结构：

```tex
\ResumeProject{项目名称（可选：准确定位）}{20XX}
\ResumeProjectSummary{一句话说明具体问题、核心 insight / 总体思路，以及可选结果。}
\ResumeProjectStack{本人实际使用并能解释的关键技术。}
\begin{ResumeBulletList}
  \item 方案如何实现，包含本人负责的对象、范围、关键机制与工程约束。
  \item 关键优化、系统接入、验证方式、完成状态或经确认结果。
\end{ResumeBulletList}
```

- `\ResumeProject` 只呈现标题和右对齐时间，不承载描述。
- `\ResumeProjectSummary` 紧接标题并单独成行，使用“具体问题—核心 insight—可选结果”建立完整因果关系，不展开实现清单。
- `\ResumeProjectStack` 放在项目介绍之后，使用较小的灰色文字；每个项目保留一行，只列本人实际使用、能够解释且与要点对应的技术。
- `ResumeBulletList` 紧接技术栈，默认两条核心亮点。只有独立高价值证据无法合并时才保留第三条；贡献较小时允许一条，不为追求完整编造内容。
- “技术栈”使用模板默认粗体作为导航；`\Key{}` 留给责任边界、核心机制、工程范围和经确认结果，每条至多一至两个短语。

## 模块选择

- 学生和应届生默认保留教育背景；学校、学院、专业和时间通常足够。
- 模块按筛选价值排序。国家级竞赛、高等级奖学金或与岗位强相关的荣誉可以紧接教育背景；普通荣誉放在项目和实习之后。
- 项目、实习、科研、开源和竞赛按岗位相关性、个人贡献和结果强度选择。
- 荣誉、专业技能和学生工作只有提供独立信息价值且页面允许时保留。
- 项目、实习、荣誉和学生工作保持独立模块，不为了节省标题混合。
- 不生成空模块，不为填满页面保留低信号内容。
- 删除完整项目、实习或奖项前先说明取舍。

## 一页排版

- 默认输出一页 A4，正文原则上不低于 10pt；
- 优先删除重复背景、空泛表达和低价值技术栈，再压缩间距或字号；
- 用有效内容撑满页面，不靠放大字号、装饰或负间距制造饱满感；
- 同类条目的标题、日期、缩进、标点和项目符号保持一致；
- 日期右对齐，换行自然，模块标题不孤立在页底；
- 页面超出时依次处理内容冗余、自然合并、模块间距、行距和页边距，最后才小幅调整字号。

## 关键内容高亮

`\Key{}` 只标记会影响招聘判断的短语：

- `独立完成`、`主导设计`、`负责核心链路`等责任边界；
- 延迟、吞吐、准确率、规模、成本或加速比等经确认指标；
- 最核心的模块、算法贡献或验证结果。

每条描述通常只高亮一至两个短语。不整句加粗，不批量加粗普通技术名词，不使用彩色底纹制造重点。

## 字体与图片

- 中文正文使用思源宋体，标题与关键短语使用对应字重的思源黑体；
- 英文、数字和西文符号使用 TeX Gyre Termes；
- 字体随模板分发，并附带各自许可证；
- 页眉支持校徽加证件照、仅校徽、仅证件照和纯文字；
- 每份简历最多一个校徽和一张证件照，图片保持纵横比；
- 草稿可以展示占位图，正式版本必须替换；用户未提供图片时将对应 `\ResumeHeader` 参数留空。

## 编译

在 Skill 根目录运行：

```bash
bash scripts/compile-resume.sh /path/to/generated-resume
```

脚本依赖 XeLaTeX 和 `pdfinfo`，编译两次后检查页数、A4 尺寸和关键日志警告。任一工具不可用时保留源文件，并明确说明未生成经过完整验证的 PDF。

## 交付检查

### 内容

- 所有事实与用户材料和纠正一致；
- 最相关、最有区分度的证据靠前；
- 强项目体现个人贡献、工程工作量和结果；
- 项目按“标题—一句话项目介绍—技术栈—两条核心亮点”排列，项目介绍说明具体问题和核心 insight；
- 数字具有口径、基线和验证方式；
- 不存在示例文字、占位数据、未解决歧义或未替换图片；
- `\Key{}` 只用于责任边界、核心贡献和确认指标。

### 文件与编译

- 输出包含 `resume.tex`、`resume.cls`、`fonts/` 和 `images/`；
- XeLaTeX 连续编译两次且没有错误；
- PDF 为一页 A4，不存在 Overfull 或 Underfull box；
- 思源宋体、思源黑体和 TeX Gyre Termes 已嵌入；
- TeX 特殊字符、网址和邮件显示正确。

### 视觉

- 姓名、联系方式和模块标题层级清楚；
- 项目、实习和教育条目格式对应；
- 页面不过空也不过度拥挤，字号可读；
- 日期、图片、正文和项目符号对齐；
- 页底无明显空洞，不存在被裁切内容或不自然换行。

文本抽取不能替代渲染后的页面检查。
